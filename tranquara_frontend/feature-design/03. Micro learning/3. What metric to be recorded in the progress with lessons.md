
## 📊 **Learning Progress Metric (Simplified Design)**

### 🎯 **Goal**

Track:
1. Which **lessons** the user has completed.
    
2. How those lessons distribute by **topic/category** for visual progress insight.
---
### 🧱 **Database Design**

#### **Table: `user_learned_lessons`**

| Column          | Type        | Description                                                                                    |
| --------------- | ----------- | ---------------------------------------------------------------------------------------------- |
| `id`            | UUID        | Primary key                                                                                    |
| `user_id`       | UUID        | From Keycloak token                                                                            |
| `collection_id` | UUID        | FK → `collections.id`                                                                          |
| `completed_at`  | TIMESTAMP   | When user finished the lesson                                                                  |
| `topic`         | VARCHAR(50) | Topic or category of the lesson (duplicate from `collections.category` for faster aggregation) |

> ✅ Each record = 1 completed lesson.

---

### ⚙️ **Collection Data Source**

- Lessons are already in `collections` with a `category` field (e.g., `"mindfulness"`, `"stress"`, `"self-awareness"`).
    
- When the user reaches the end of a lesson’s slide flow → backend creates a `user_learned_lessons` record.

### 🧠 **AI Personalization Usage**

The `user_learned_lessons` data can also guide your recommendation model:

- Avoid repeating lessons from the same `collection_id`.
    
- Balance underrepresented categories:
    
    > “You’ve focused mostly on mindfulness lately — want to explore emotional awareness next?”
